How It Works

  • Each user can claim one coupon at a time
  • You must wait 1 hour before claiming another coupon
  • Coupons are distributed in a round-robin fashion
  • No login required - we use cookies to track your session

What I Have Used

This coupon distribution system is built using modern web technologies and serverless architecture:

  • Frontend: HTML5, CSS3, and vanilla JavaScript for a lightweight, responsive user interface
  • Backend: Netlify Functions (serverless) for handling coupon distribution logic
  • Database: FaunaDB for storing coupons and user claims in a serverless, globally distributed database
  • User Tracking: HTTP cookies and browser localStorage for identifying users without requiring login
  • Distribution Algorithm: Round-robin system that cycles through available coupons fairly
  • Security: Rate limiting (1 hour cooldown) to prevent abuse and ensure fair distribution
  • Deployment: Netlify for continuous deployment and hosting of both static assets and serverless functions

The system maintains two collections in FaunaDB: "coupons" for storing available coupon codes, and "claims" for tracking which users have claimed which coupons and when. When a user claims a coupon, it's moved to the end of the queue (round-robin) and a claim record is created with a timestamp to enforce the cooldown period.
